## What is css-variants?

**css-variants** is a zero-dependency, type-safe library for managing CSS class variants in JavaScript and TypeScript. Define style variations (color, size, state) declaratively, and let the library generate the correct CSS classes at runtime.

**Key facts:**
- **~1KB gzipped** — smallest variant library available
- **3-11x faster** — outperforms CVA and tailwind-variants
- **Zero dependencies** — no clsx, no tailwind-merge bundled
- **Framework-agnostic** — works with React, Vue, Svelte, Solid, vanilla JS
- **CSS-agnostic** — works with Tailwind, CSS Modules, vanilla CSS, CSS-in-JS

<Aside type="tip" title="Alternative to CVA and tailwind-variants">
If you're using CVA (Class Variance Authority) or tailwind-variants, css-variants is a drop-in replacement with better performance and additional features like slot variants and style variants.
</Aside>

## Quick Example

```typescript
import { cv } from 'css-variants'

const button = cv({
  base: 'px-4 py-2 rounded font-medium transition-colors',
  variants: {
    color: {
      primary: 'bg-blue-600 text-white hover:bg-blue-700',
      secondary: 'bg-gray-200 text-gray-800 hover:bg-gray-300',
    },
    size: {
      sm: 'text-sm px-3 py-1.5',
      lg: 'text-lg px-6 py-3',
    },
  },
  defaultVariants: {
    color: 'primary',
    size: 'sm',
  },
})

button()                     // Primary + Small (defaults)
button({ color: 'secondary', size: 'lg' }) // Secondary + Large
```

## Works Without Tailwind CSS

css-variants is **not tied to Tailwind CSS**. Use it with any CSS approach:

```typescript
// Tailwind CSS
const button = cv({
  base: 'px-4 py-2 rounded',
  variants: { color: { primary: 'bg-blue-600 text-white' } },
})

// Vanilla CSS
const button = cv({
  base: 'btn',
  variants: { color: { primary: 'btn-primary' } },
})

// CSS Modules
import styles from './Button.module.css'
const button = cv({
  base: styles.btn,
  variants: { color: { primary: styles.primary } },
})
```
